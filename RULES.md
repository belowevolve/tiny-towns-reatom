# Tiny Towns — Правила игры

Структурированные правила для использования как источник истины при разработке.

---

## R1. Базовые понятия

| ID   | Понятие       | Значение                                                   |
| ---- | ------------- | ---------------------------------------------------------- |
| R1.1 | Сетка         | Планшет 4×4 = 16 клеток                                    |
| R1.2 | Клетка        | Может содержать: ничего (пустая), один ресурс, одно здание |
| R1.3 | Ресурсы       | 5 типов: дерево, пшеница, кирпич, стекло, камень           |
| R1.4 | Здания        | Определяются картами (набор карт выбирается перед игрой)   |
| R1.5 | Коттедж       | Всегда в игре (не выбирается)                              |
| R1.6 | Монументы     | Уникальные здания — у каждого игрока свой (не реализовано) |
| R1.7 | Соседство     | Клетки по горизонтали/вертикали. Диагональ НЕ считается    |
| R1.8 | Победные очки | VP. Могут быть отрицательными                              |

## R2. Подготовка к игре

| ID   | Шаг                                                                            |
| ---- | ------------------------------------------------------------------------------ |
| R2.1 | Каждый игрок получает планшет 4×4                                              |
| R2.2 | Карта коттеджа — всегда в игре                                                 |
| R2.3 | Из каждой категории зданий берётся по 1 карте (случайно или по договорённости) |
| R2.4 | Остальные карты убираются — в этой партии не используются                      |
| R2.5 | Кубы ресурсов и фишки зданий бесконечны (общий запас)                          |
| R2.6 | Первый главный строитель — тот, кто последним строил в реальной жизни          |

## R3. Ход игры (раунд)

```
РАУНД:
  1. Главный строитель объявляет тип ресурса
  2. ВСЕ игроки ОБЯЗАНЫ взять куб этого ресурса и поставить на пустую клетку
  3. Игроки МОГУТ построить здания (необязательно)
  4. Молот переходит к следующему игроку (по часовой стрелке)
```

| ID    | Правило                                                                             |
| ----- | ----------------------------------------------------------------------------------- |
| R3.1  | Главный строитель объявляет ОДИН тип ресурса                                        |
| R3.2  | ВСЕ игроки одновременно берут куб этого типа                                        |
| R3.3  | Каждый игрок ОБЯЗАН поместить куб на любую пустую клетку своего планшета            |
| R3.4  | Размещённые ресурсы НЕЛЬЗЯ двигать                                                  |
| R3.5  | Единственный способ убрать ресурс — построить из него здание                        |
| R3.6  | После размещения ресурса игрок МОЖЕТ построить здания (0, 1 или несколько)          |
| R3.7  | Строить можно ТОЛЬКО после размещения ресурса, но ДО объявления следующего типа     |
| R3.8  | Не обязательно строить здание сразу — можно отложить на будущие раунды              |
| R3.9  | Можно построить НЕСКОЛЬКО зданий за один раунд                                      |
| R3.10 | Эффекты зданий применяются только после того, как все игроки разместили свои здания |
| R3.11 | Новый раунд начинается только когда ВСЕ игроки готовы                               |

## R4. Строительство зданий

| ID   | Правило                                                                              |
| ---- | ------------------------------------------------------------------------------------ |
| R4.1 | Схемы зданий можно ПОВОРАЧИВАТЬ и ОТРАЖАТЬ любым способом                            |
| R4.2 | Должна совпадать структура расположения ресурсов И их типы                           |
| R4.3 | Убрать с планшета подходящие ресурсы (вернуть в общий запас)                         |
| R4.4 | Фишка здания ставится на ОДНУ из клеток, откуда только что убрали ресурс             |
| R4.5 | Каждый ресурс используется для постройки только одного здания                        |
| R4.6 | Здание НЕЛЬЗЯ перемещать после постройки                                             |
| R4.7 | Можно построить любое здание из текущего набора карт (неограниченное количество раз) |
| R4.8 | Карты зданий доступны до конца игры, не сбрасываются после постройки                 |

## R5. Снабжение коттеджей (feeding)

| ID   | Правило                                                                            |
| ---- | ---------------------------------------------------------------------------------- |
| R5.1 | Коттедж приносит 3 VP ТОЛЬКО если его снабжает здание с символом «кормилец»        |
| R5.2 | Кормилец = здание со спец. символом (🍴). НЕ все здания являются кормильцами       |
| R5.3 | Кормилец снабжает коттеджи согласно своей механике (Ферма — в любом месте города)  |
| R5.4 | У каждого кормильца есть ЛИМИТ — максимальное число коттеджей, которые он снабжает |
| R5.5 | Неснабженные коттеджи приносят 0 VP, но остаются на поле                           |
| R5.6 | Некоторые здания (напр. колодец) считают коттеджи НЕЗАВИСИМО от снабжения          |

## R6. Завершение игры

| ID   | Правило                                                                                  |
| ---- | ---------------------------------------------------------------------------------------- |
| R6.1 | Когда город полностью заполнен ресурсами и игрок не может/не хочет строить — он выбывает |
| R6.2 | Можно продолжать играть, пока есть свободные клетки, даже если нечего строить            |
| R6.3 | Если все противники выбыли — продолжаете играть один, пока город не заполнен             |
| R6.4 | Игра заканчивается когда ВСЕ игроки завершили строительство                              |

## R7. Подсчёт очков

| ID   | Правило                                                         |
| ---- | --------------------------------------------------------------- |
| R7.1 | Убрать с планшета ВСЕ оставшиеся кубы ресурсов                  |
| R7.2 | Каждая пустая клетка (включая те, где были ресурсы) = **-1 VP** |
| R7.3 | Каждое здание приносит VP согласно своей карте                  |
| R7.4 | Итог = сумма VP зданий - штрафы за пустые клетки                |
| R7.5 | Можно набрать отрицательное итоговое количество VP              |

## R8. Определение победителя

| ID   | Приоритет | Критерий                            |
| ---- | --------- | ----------------------------------- |
| R8.1 | 1         | Больше всего VP                     |
| R8.2 | 2         | Реже был главным строителем         |
| R8.3 | 3         | Меньше пустых клеток на планшете    |
| R8.4 | 4         | Больше коттеджей                    |
| R8.5 | 5         | Ничья — побеждают несколько игроков |

## S. Одиночный режим

| ID  | Правило                                                        |
| --- | -------------------------------------------------------------- |
| S1  | Используется колода карт ресурсов (15 карт: по 3 каждого типа) |
| S2  | Перемешать колоду, вскрывать верхнюю карту каждый раунд        |
| S3  | Игрок обязан разместить объявленный ресурс                     |
| S4  | Цель — набрать как можно больше VP                             |
